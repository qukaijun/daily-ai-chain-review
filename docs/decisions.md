# Decisions

## 2026-06-21

- 新项目采用兄弟项目方式，不塞进“每日大A板块报告”原仓库。
- 复用大盘日报的工程思路：入口脚本、输出目录、HTML 渲染、健康检查、截图脚本、GitHub 流程。
- 第一版 MVP 使用结构化 JSON 输入，不先做自动爬取。
- 研报和小作文纳入事件系统，但按证据等级约束写入层级。
- 小作文/传闻进入验证池，不单独改变核心假设。
- 工具定位为研究辅助和复盘，不构成投资建议，不输出自动交易指令。
- 研报、小作文和公告统一用 JSON 模板导入；模板放在 `data_sources/_templates/`，不参与报告生成。
- 公告目录纳入事件加载链路；小作文和非高等级证据不得设置 `model_update_candidate=true`。
- 数据源接入采用 provider 管理器：每个来源保留状态、错误、获取时间和证据层级；新闻/搜索/行情类数据只进入候选或事件层，不直接改核心假设。
- 复用大盘日报的密钥管理方式：读取全局 `E:\AI工具\secrets\llm.env` 和项目 `.env.local`，不把 API key 写入仓库或聊天。
- Perplexity 作为独立搜索增强源，优先请求结构化事件和引用；失败时回退摘要解析，结果默认进入 `search_api` 与验证池。
- 小作文、传闻、新闻和搜索增强事件采用验证状态生命周期：`pending/confirmed/rejected/expired/upgraded/not_required`。
- 公告索引采用 AkShare `stock_notice_report` 拉取东方财富公告大全，命中 AI 股票池或 AI 关键词后作为 `company_announcement` 高等级证据候选。
- 低证据事件命中同个股公告时只标记“已找到公告候选”，不自动认定原事件已验证，也不自动修改核心假设。
- 公告详情抓取使用东方财富公告正文接口，保留正文摘要、PDF 链接、金额/比例/日期标记和人工复核清单；这些字段只提升复核效率，不替代人工阅读原文。
- 公告候选的人工确认采用独立 `data_sources/verifications/` 写回层，按 `event_id` 覆盖验证状态并保留复核人、证据链接和决策说明；原始事件文件不被自动改写。
- `model_update_candidate=true` 只表示进入核心假设复核候选，必须连到公告、交易所文件或财报等高等级证据，不代表系统自动修改估值模型。
- 项目路线转为全自动优先：不继续建设研报手工导入助手，改为 `perplexity_research` 与 `perplexity_rumors` 自动增强源；自动研报和传闻默认进入事件层/验证池，不自动改变核心假设。
- 自动验证评分与去重采用报告期内临时聚类，不改写原始事件；分数用于复核排序和重复提示，不代表系统自动确认事实。
- TradingAgents 式多角色层先采用本地确定性规则实现，确保无 LLM key 时也能稳定生成；角色输出只提供研究观察、依据、风险和下一步验证，不输出自动交易建议。
- 每日自动化先落在仓库内脚本：`run_daily_review.py` 负责生成、日志和自检，`check_latest_run.py` 负责产物巡检，`install_daily_task.ps1` 负责 Windows 交易日定时；Codex 定时器作为通知/异常诊断层，不替代项目自身脚本。
- LLM 深度多角色复盘采用显式开启策略：默认不调用模型，`--deep-agents` 或 `DAA_ENABLE_DEEP_AGENTS=1` 才启用；无 key 或调用失败时降级为本地确定性层，且不改变证据等级、验证状态、核心假设或投资结论边界。
- 外部通知通道采用显式开启策略：默认仅 console 预览，`--notify` 加 `DAA_NOTIFY_ENABLED=1` 才发送；首版支持企业微信 webhook，通知只发紧凑摘要和本地报告路径，不发送报告全文、API key 或 webhook 明文。
- 通知分级采用 `success/warning/failure`：可用但有 provider 异常时不阻断日报，而以 warning 摘要提示；自动化命令失败时读取 `latest_run.json` 和日志尾部生成 failure 告警，便于定位失败命令。
- 通知发送记录采用 JSONL 运行产物存放在 `output_files/notification_logs/`；真实 webhook 发送支持有限重试和指数退避，记录只保存脱敏 webhook、状态、尝试次数、错误摘要和报告路径，不提交 GitHub。
- 巡检和通知按“最近已完成交易日”选择目标复盘日，默认盘后时间为 17:00；周末、节假日和盘前使用上一交易日，避免午夜测试产物覆盖正式市场源日报。节假日和补班交易日先用环境变量维护，后续可升级为年度日历文件或官方数据源。
- 任务板历史项按实际路线收口：手工研报导入助手由自动研报增强源替代，小作文验证追踪由验证状态生命周期和人工确认写回覆盖，多角色分析由 N22 完成。
- A股交易日历升级为仓库内年度 JSON 文件，首版使用上交所 2025-12-22 发布的 2026 年部分节假日休市安排；环境变量继续作为临时补充覆盖，不替代可审计年度文件。

## 2026-06-22

- 客户交付路线先做 V1 绿色免安装包：客户不需要 Codex，解压后通过 bat 脚本安装依赖、检查环境、生成复盘、导出单文件 HTML 和导出诊断材料。
- 客户包不内置 API Key，不复制 `.env`、`.env.local`、`secrets.env`、`llm.env`、旧报告、运行 JSON、Codex 线程或 Git 仓库。
- 客户包启动脚本默认使用包内 `secrets/llm.env` 作为可选密钥路径，同时仍支持客户复制 `.env.local.example` 为 `.env.local`。
- 客户诊断分为必需检查和可选能力检查：Python、健康检查和密钥泄漏扫描失败才阻断；Perplexity、LLM 深度层、外部数据源未配置只作为可选能力告警。
- 第一版商业边界仍是研究复盘和服务交付，不承诺收益、不做自动交易、不把开源底座本身包装成封闭产品。
